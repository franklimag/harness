# 研究功能学习笔记

> hermes-agent 的研究级特性 — 轨迹生成、轨迹压缩与 DSPy+GEPA 自进化。

## 概述

hermes-agent 区别于纯工具性 Agent 的关键特征之一是其研究级功能。它能生成训练数据、压缩轨迹用于模型微调，并通过 DSPy+GEPA 实现 prompt 自进化。

## 轨迹生成 (Trajectory Generation)

### 什么是轨迹

```
轨迹 (Trajectory) = Agent 完成任务的完整执行记录

包含:
  - 用户请求 (input)
  - Agent 思考过程 (reasoning)
  - 工具调用序列 (actions)
  - 工具返回结果 (observations)
  - 最终输出 (output)
  - 元数据 (模型/时间/成功与否)
```

### Batch 模式

```
批量轨迹生成:

输入: 任务列表 (JSON/YAML)
  tasks:
    - "实现一个 LRU 缓存"
    - "修复这个 SQL 注入漏洞"
    - "将这个类重构为函数式风格"
    ...

执行: Agent 逐个执行任务，记录完整轨迹

输出: 轨迹数据集
  trajectories/
    ├── task_001.jsonl  (LRU 缓存)
    ├── task_002.jsonl  (SQL 注入)
    └── task_003.jsonl  (重构)
```

### 轨迹用途

| 用途 | 说明 |
|------|------|
| 模型微调 | 用高质量轨迹训练专门化模型 |
| 评估 | 对比不同模型/配置的表现 |
| 教学 | 展示问题的最佳解决路径 |
| 调试 | 分析 Agent 失败的原因 |

## 轨迹压缩 (Trajectory Compression)

### 为什么需要压缩

```
原始轨迹问题:
  - 包含大量冗余步骤 (试错、重复查看)
  - Token 数过多，训练成本高
  - 噪声信息降低训练效果

压缩后:
  - 只保留关键决策步骤
  - 去除冗余和试错
  - 训练数据质量更高
  - 训练成本更低
```

### 压缩策略

```
原始轨迹 (20 步):
  1. 读文件 A
  2. 读文件 B  
  3. 读文件 A (重复)     ← 去除
  4. 尝试方案 1 (失败)   ← 可去除
  5. 尝试方案 2 (失败)   ← 可去除
  6. 分析错误原因
  7. 尝试方案 3 (成功)
  8-20. ...

压缩后 (8 步):
  1. 读文件 A, B (合并)
  2. 分析 → 选择方案 3 (跳过试错)
  3-8. 关键执行步骤
```

### 压缩级别

| 级别 | 保留内容 | 用途 |
|------|---------|------|
| L1 (轻度) | 去除明显重复 | 完整分析 |
| L2 (中度) | 合并相似步骤 | 高效训练 |
| L3 (重度) | 只留关键决策 | 蒸馏/摘要 |

## DSPy + GEPA 自进化

### DSPy (Declarative Self-Improving Python)

```
DSPy 的核心思想:
  不手工写 prompt
  而是定义"输入→输出"的规格
  让系统自动优化 prompt

在 hermes-agent 中:
  定义: "给定用户请求，生成最佳工具调用序列"
  DSPy: 自动优化调用策略的 prompt
```

### GEPA (Genetic Evolution of Prompt Agents)

```
GEPA 的核心思想:
  用遗传算法进化 Agent 的行为

  种群 = 多个 prompt 变体
  适应度 = 任务成功率
  进化 = 交叉 + 变异 + 选择

  多代进化后 → 自然选择出最优 Agent 行为
```

### 自进化流程

```
┌─────────────────────────────────────────────┐
│  1. 生成轨迹 (当前 Agent 执行任务)          │
│  2. 评估质量 (成功率/效率/Token 消耗)       │
│  3. DSPy 优化 (改进 prompt)                │
│  4. GEPA 进化 (群体级优化)                  │
│  5. 部署改进版 Agent                        │
│  6. 重复 (持续进化)                         │
└─────────────────────────────────────────────┘
```

## hermes-agent-self-evolution 分支

| 组件 | 技术 | 功能 |
|------|------|------|
