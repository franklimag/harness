# 闭环学习系统学习笔记

> hermes-agent 核心差异化 — 自进化 Skills、记忆系统、FTS5 搜索与 Honcho 用户建模。

## 概述

hermes-agent 的闭环学习 (Closed Learning Loop) 是其最核心的差异化特性。Agent 不是静态的工具，而是"越用越好"的进化系统。

## Skills 自创建与自改进

### 从经验创建 Skill

```
场景: 用户多次要求"在提交前运行 lint"

Agent 观察:
  第 1 次: 用户手动提醒运行 lint
  第 2 次: 用户又提醒了
  第 3 次: Agent 意识到这是一个模式

Agent 学习:
  创建 Skill: "提交前自动运行 lint"
  触发条件: git commit 前
  动作: 运行 npm run lint

下次使用:
  Agent 自动在 commit 前运行 lint，无需用户提醒
```

### Skill 使用中改进

```
Skill v1: "提交前运行 lint"
  使用 3 次后发现: 用户有时 --fix 有时不 --fix

Skill v2 (自动改进): "提交前运行 lint --fix，如果有无法自动修复的问题则报告"
  观察: 用户每次看到可修复问题都会选 --fix

Skill v3 (继续改进): "提交前运行 lint --fix，只报告严重问题"
  观察: 用户忽略 warning 级别的报告
```

## 记忆系统架构

### 三层记忆

| 层 | 技术 | 内容 | 检索方式 |
|---|------|------|---------|
| Agent 策展 | 结构化存储 | 关键决策、偏好、约束 | 主动推送 |
| FTS5 搜索 | SQLite FTS5 | 完整会话历史 | 全文检索 |
| Honcho 模型 | 辩证法建模 | 用户行为模式、偏好 | 自动应用 |

### Agent 策展记忆

```
Agent 策展 = Agent 主动决定"什么值得记住"

不是记录所有对话 (那太多了)
而是 Agent 判断后保存:
  - "用户偏好 TypeScript 而非 JavaScript"
  - "这个项目使用 pnpm 而非 npm"
  - "用户喜欢简洁的 commit message"
```

### 知识持久化提示 (Nudge)

```
Agent 发现可能有价值的知识:
  │
  ├── Agent: "我注意到你总是用 --strict 模式运行 TypeScript。
  │           要我记住这个偏好吗？"
  │
  ├── 用户: "对，记住"
  │
  └── Agent: 持久化 → 下次自动使用 --strict

= "Agent 提醒用户持久化知识"
= 不强制记忆，用户决定
```

### FTS5 全文搜索

```
SQLite FTS5 索引:
  - 所有会话的文本内容
  - 支持: 精确匹配 / 前缀匹配 / 近义词
  - 查询: "上次我们讨论过 API 限速吗?"
  - 返回: 相关会话片段 + 上下文

优势:
  - 轻量 (无需外部搜索服务)
  - 快速 (SQLite 原生)
  - 离线可用
  - 增量更新
```

## Honcho 辩证法用户建模

### 核心概念

```
传统用户模型:
  key-value 存储: { language: "TypeScript", style: "concise" }
  问题: 扁平，无法捕捉矛盾和变化

Honcho 辩证法:
  正题 (Thesis): "用户倾向简洁代码"
  反题 (Antithesis): "但在复杂逻辑处偏好详细注释"
  合题 (Synthesis): "用户偏好代码简洁 + 关键处注释"
  
  = 能捕捉用户行为中的矛盾与统一
```

### 建模维度

| 维度 | 示例 |
|------|------|
| 编码风格 | 简洁 vs 详尽 (视情况) |
| 决策模式 | 快速原型 vs 仔细规划 (视复杂度) |
| 沟通偏好 | 简短回复 vs 详细解释 (视主题) |
| 技术偏好 | 成熟方案 vs 新技术 (视风险) |
