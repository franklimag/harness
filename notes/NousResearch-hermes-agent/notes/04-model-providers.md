# 模型提供商与模型无关设计学习笔记

> hermes-agent 的模型层 — 10+ 提供商、200+ 模型、回退链与动态切换。

## 概述

hermes-agent 实现了真正的模型无关设计，支持 10+ 模型提供商和 200+ 模型。通过统一接口和回退链，确保服务高可用和成本最优。

## 回退链设计

```
回退链示例:

优先级 1: Nous Portal (自有，最快)
    │ 不可用/限速
    ▼
优先级 2: OpenRouter → Claude Sonnet (性能好)
    │ 不可用/超预算
    ▼
优先级 3: NovitaAI → Llama 405B (性价比)
    │ 不可用
    ▼
优先级 4: HuggingFace → 本地模型 (兜底)

每次请求自动选择最高优先级可用的提供商
```

### 回退触发条件

| 条件 | 说明 |
|------|------|
| API 不可用 | 连接失败/超时 |
| 限速 (Rate Limit) | 达到请求限制 |
| 超预算 | 超过配置的成本阈值 |
| 模型不可用 | 特定模型下线/维护 |
| 质量不满足 | 响应质量低于阈值 |

## 模型选择策略

```
简单任务 (格式化/重命名):
  → 小模型 (Haiku/小型 Llama)
  → 成本最低

中等任务 (编码/调试):
  → 中模型 (Sonnet/Llama 70B)
  → 平衡成本和质量

复杂任务 (架构/推理):
  → 大模型 (Opus/Llama 405B)
  → 质量优先
```
